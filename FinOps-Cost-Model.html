4. FinOps Optimization Levers

Bronze Data Retention Policy — Raw Kafka events in Bronze are the largest data volume. Retain for 90 days only. After 90 days, Silver and Gold datasets are the system of record. Archive to S3 Glacier ($0.004/GB/month) for regulatory compliance if needed. This alone can reduce Foundry storage costs 40–60%.
Compute Right-Sizing — Not all pipeline transforms need large Spark clusters. Bronze→Silver transforms for low-volume domains (Vibration, Acoustic) can run on small Foundry compute profiles. Reserve large compute for Silver→Gold aggregations and ML training. Review compute profile assignments quarterly.
Minimize Full Foundry Seats — Foundry named user licenses are expensive. Data engineers and data scientists need full seats. Business analysts and dealers do NOT — they use Foundry Contour (read-only dashboards) or Member Portal (application layer). Audit user types annually and downgrade where full seats are unused.
Kafka MSK Tiered Storage — Instead of retaining 7 days of Kafka events on expensive broker disk, enable MSK Tiered Storage. Older messages moved to S3 at $0.023/GB vs. EBS disk at $0.10/GB. 4× storage cost reduction for Kafka long retention.
API Gateway Caching — Enable API Gateway response cache for GET /machines/{serial} with 5-minute TTL (aligns with Redis TTL). Reduces Foundry Ontology query volume by ~70% for dashboard refresh patterns (most Portal users refresh the same machine view repeatedly).
Pipeline Scheduling Optimization — Not all Silver→Gold transforms need to run hourly. Non-critical domains (Vibration, Acoustic, Structural) can run every 4 hours with no business impact. Only Engine, DPF/DEF, Hydraulics, Electrical need hourly Gold refresh. Reducing compute schedule for 7 of 12 domains saves ~35% on pipeline compute.
ML Training Cadence — Weekly retraining is sufficient for NBO propensity models (machine behavior patterns don't change faster than weekly). Avoid over-training (daily batch) — this adds compute cost with diminishing accuracy improvement.
