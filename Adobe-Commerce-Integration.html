Adobe Commerce Integration

Parts Store — NBO Deep Link, Cart Pre-population & Order Closed Loop

Prepared by Harsh Shah  |  May 2026

1. Integration Overview

Three integration touchpoints between the Machine Telemetry Platform and Adobe Commerce

Key design decision: Adobe Commerce contract pricing is called live at serve time by the NBO Engine — not cached in Foundry Gold. Part prices change frequently (contract tiers, promotions). Caching stale prices in recommendations would break customer trust. All other NBO data (urgency score, part compatibility, ranking) comes from Foundry Gold.

2. Integration Point 1 — NBO Deep Link to Pre-populated Cart

How the Deep Link Works

1
NBO Engine runs — reads Gold layer (urgency score, wear%, fault severity, component compatibility) and ranks recommended parts for each machine.
2
Contract Pricing API called — NBO Engine calls Adobe Commerce REST API (GET /V1/products/{sku}/pricing?customer_group={tier}) for each top-ranked part to retrieve live contract price for this customer's tier.
3
Deep link constructed — NBO Engine builds a signed deep link URL that encodes the Adobe Commerce cart payload. URL is signed with HMAC-SHA256 to prevent tampering.
4
NBO payload delivered to Member Portal — Portal renders NBO widget with part cards showing urgency badge, part name, compatibility, and contract price.
5
Customer clicks Add to Cart — Browser follows deep link to Adobe Commerce. Commerce validates HMAC, creates pre-populated cart with the specified SKU, quantity, and customer's contract price applied automatically.
6
Customer completes checkout — Standard Adobe Commerce checkout flow. Customer can modify quantity or remove items but cannot change the contract price applied.

Deep Link URL Structure

https://parts.cat.com/cart/add?
  sku=CAT-1R-0750
  &qty=1
  &customer_group=FLEET_GOLD
  &machine_serial=CAT-EX-2024-00142
  &nbo_id=nbo_abc123
  &urgency=HIGH
  &source=member_portal
  &expires=1747000000
  &sig=HMAC-SHA256(secret, sku+qty+customer_group+machine_serial+nbo_id+expires)
Security: The expires timestamp limits link validity to 24 hours. The HMAC sig prevents parameter tampering (e.g., changing customer_group to get a better price tier). Adobe Commerce validates sig before creating the cart session.

3. Integration Point 2 — Adobe Commerce Contract Pricing API

API Call at Serve Time

DetailValue
EndpointGET /rest/V1/products/{sku}/tier-prices?customer_group_id={group_id}
AuthOAuth 2.0 Client Credentials. NBO Engine holds client_id/client_secret in AWS Secrets Manager. Token TTL: 1 hour, cached by NBO Engine.
Call timingCalled per-part at NBO generation time. Results NOT cached in Foundry (live price only). Timeout: 2 seconds. Fallback: use list_price with disclaimer "contract price pending".
Customer group mappingCustomer tier from Foundry Ontology Customer object (contract_tier property, sourced from MDM) maps to Adobe Commerce customer_group_id via a lookup table in Foundry Silver.
Inventory checkSeparate call: GET /rest/V1/stockItems/{sku}. Returns qty and availability status. NBO Engine shows "IN_STOCK", "LOW_STOCK (<5)", or "ORDER_REQUIRED" on the part card.
Rate limit handlingAdobe Commerce API rate limit: 1000 req/min. NBO Engine batches pricing calls (up to 20 SKUs per machine recommendation set). Circuit breaker: if 3 consecutive timeouts, fall back to list price for 5 minutes.

4. Integration Point 3 — Order Webhook (Closed Loop)

Adobe Commerce notifies Foundry on every order fulfillment — closing the feedback loop for NBO model retraining

Webhook Flow

1
Order placed in Adobe Commerce — Customer completes checkout. Adobe Commerce creates order record.
2
Webhook fired on order_complete event — Adobe Commerce POSTs signed payload to Foundry webhook endpoint within 30 seconds of fulfillment status.
3
Foundry validates HMAC signature — Webhook receiver Lambda validates X-Commerce-Signature header. Rejects if invalid or timestamp older than 5 minutes.
4
Payload written to Bronze webhook dataset — Raw order event appended to /bronze/adobe_commerce_orders. Idempotency key: order_id — duplicate webhooks are deduplicated.
5
Silver/Gold pipeline enriches order — Joins to NBO_Recommendation object via nbo_id to mark recommendation as ACTIONED. Updates Machine Ontology object with last_parts_order_date.
6
NBO suppression updated — Ordered part triggers 30-day suppression rule in NBO Engine (don't recommend the same part again within 30 days of purchase).
7
Model retraining signal — Weekly ML retraining job reads actioned vs. dismissed NBO recommendations to update propensity model weights.

6. Error Handling & Resilience

Failure ScenarioBehaviorUser Impact
Adobe Commerce API timeout (pricing call >2 sec)Fallback to list_price with label "Estimated price — contract price applied at checkout". Circuit breaker opens after 3 failures.Customer sees list price on card, actual contract price applied at checkout. No blocked experience.
Adobe Commerce API downNBO recommendations shown without pricing. "Price available at checkout" label. Deep link still generated with SKU — Adobe Commerce shows price at cart.NBO widget still loads. Customer can still click through. Degraded pricing display only.
Deep link HMAC validation fails (at Adobe Commerce)Adobe Commerce returns 403. Member Portal shows "Link expired — please refresh" and calls NBO Engine to regenerate fresh deep link.Customer sees one retry prompt. Second deep link generated within 2 seconds.
Order webhook delivery fails (Foundry endpoint down)Adobe Commerce retries webhook: immediately, then +1 min, +5 min, +30 min, +2 hrs. Dead letter after 5 failed attempts — ops alert triggered.Order is fulfilled normally. Foundry feedback loop delayed by up to 2 hours. NBO suppression delayed.
Duplicate webhook (same order_id received twice)Foundry deduplicates on order_id before writing to Bronze. Second webhook silently dropped.No impact. Idempotent design.
SKU not in Foundry parts catalogNBO suppression rule fires — recommendation not generated for this component. Logged to data quality alert queue.Customer does not see a recommendation for that part. Parts catalog team alerted to add mapping.
