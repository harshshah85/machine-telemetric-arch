3. Link Type Definitions

Link Types define relationships between Object Types. Each link has a direction, cardinality, and a label describing the relationship.
Link TypeFrom ObjectTo ObjectCardinalityDescription
CONTAINSFleetMachineOne-to-ManyA fleet contains one or more machines
OWNSCustomerMachineOne-to-ManyA customer owns one or more machines
SERVED_BYCustomerDealerMany-to-OneA customer is assigned to a dealer
OPERATESOperatorMachineMany-to-ManyAn operator can run multiple machines; a machine can have multiple operators
HOSTSJobSiteMachineOne-to-ManyA job site hosts one or more machines (geofence-based assignment)
HAS_COMPONENTMachineComponentOne-to-ManyA machine has multiple components across all telemetry domains
GENERATESMachineTelemetryEventOne-to-ManyA machine generates continuous telemetry events (time-series)
TRIGGERS_ALERTMachineAlertOne-to-ManyMachine-level fault codes trigger alerts
COMPONENT_ALERTComponentAlertOne-to-ManyComponent-level threshold breach triggers alert
REQUIRES_PARTComponentPartMany-to-OneA component maps to its replacement OEM part
HAS_WORKORDERMachineWorkOrderOne-to-ManyA machine has a history of work orders
ACTIONED_BYAlertWorkOrderMany-to-OneAn alert triggers the creation of a work order
SERVICED_BYWorkOrderDealerMany-to-OneA work order is fulfilled by a dealer
USES_PARTWorkOrderPartMany-to-ManyA work order uses one or more parts
HAS_NBOMachineNBO_RecommendationOne-to-ManyA machine has active NBO recommendations
HAS_NBAMachineNBA_RecommendationOne-to-ManyA machine has active NBA recommendations
LINKS_PARTNBO_RecommendationPartMany-to-OneAn NBO recommendation links to a specific part in the catalog
CUSTOMER_NBOCustomerNBO_RecommendationOne-to-ManyCustomer-scoped NBO view (all machines)
CUSTOMER_NBACustomerNBA_RecommendationOne-to-ManyCustomer-scoped NBA view (all machines)

5. Action Type Definitions

Action Types define operations that can be performed on objects — either by users via the portal or triggered automatically by the NBO/NBA engine.
Action TypeOn ObjectTriggered ByWhat It Does
RAISE_ALERTMachine / ComponentTelemetry threshold breachCreates new Alert object, sets severity, triggers notification pipeline
CREATE_WORKORDERAlertUser or NBA engineCreates WorkOrder linked to Alert, assigns to dealer, sets status = OPEN
CLOSE_WORKORDERWorkOrderDealer confirmation webhookSets status = COMPLETED, resets Component wear %, triggers model retraining
GENERATE_NBOMachineFoundry AIP pipeline (daily)Scores components, ranks parts, writes NBO_Recommendation objects
GENERATE_NBAMachineFoundry AIP pipeline (real-time + daily)Scores alerts + usage patterns, writes NBA_Recommendation objects
ORDER_PARTSNBO_RecommendationCustomer via portal → Adobe CommerceRedirects to Adobe Commerce cart, updates NBO status = ORDERED on webhook
BOOK_SERVICENBA_RecommendationCustomer via portalDeep links to dealer scheduling, creates WorkOrder on confirmation
DISMISS_RECOMMENDATIONNBO / NBACustomer or Fleet ManagerSets status = DISMISSED, feeds back into suppression logic
SEND_NOTIFICATIONAlert / NBASeverity = CRITICAL auto-triggerPushes SMS/email to operator, fleet manager, and dealer
RESET_COMPONENT_WEARComponentWorkOrder CLOSE_WORKORDERResets wear_percentage and last_replaced_date after part replacement
RETRAIN_NBO_MODELNBO ModelORDER_PARTS webhook (weekly batch)Incorporates new purchase data into propensity model
RETRAIN_NBA_MODELNBA ModelCLOSE_WORKORDER (weekly batch)Incorporates service outcomes into failure prediction model

6. IAM — Object-Level Access Control

Palantir Foundry enforces row-level security via Data Markings. Each object is tagged with a marking that restricts visibility to authorized IAM roles only.
